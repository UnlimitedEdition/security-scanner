# Code of Conduct

## Our Pledge

We are committed to making **Web Security Scanner** an open, welcoming,
and harassment-free project for everyone — regardless of experience level,
gender, gender identity and expression, sexual orientation, disability,
personal appearance, body size, race, ethnicity, age, religion, nationality,
or technical background.

Security tooling improves when diverse perspectives contribute to it.
People must feel comfortable reporting bugs, asking questions, and
disagreeing about design choices.

## Expected Behavior

- **Focus on the work.** Technical disagreement is welcome; personal
  attacks are not. Critique code and ideas, not the contributor behind them.
- **Be patient with newcomers.** Everyone started somewhere. A thoughtful
  answer to a "basic" question strengthens the whole project.
- **Give credit.** If someone's idea, PR, or bug report helped your work,
  acknowledge it in the commit message or PR description.
- **Respect operator context.** Many users of this scanner are non-technical
  website owners. Documentation, error messages, and issue responses should
  meet people where they are.
- **Assume good intent.** Written communication lacks tone. If a message
  reads ambiguously, assume the charitable interpretation first.
- **Stay on topic.** Project spaces are for project work. Off-topic
  discussions, promotional content, and political debates belong elsewhere.

## Unacceptable Behavior

The following behaviors are not tolerated in any project space:

- **Harassment** of any kind — including but not limited to offensive
  comments, deliberate intimidation, stalking, unwelcome sexual
  attention, or sustained disruption of discussions
- **Discrimination** — excluding people or treating them unfairly based
  on any protected characteristic
- **Doxxing** — publishing someone's private information without consent
- **Trolling** — deliberately provoking emotional responses or derailing
  constructive conversation
- **Professional dishonesty** — misrepresenting credentials, plagiarizing
  code, or claiming credit for others' work

In short: treat others the way a competent colleague would treat you on
their best day.

## Scope

This Code of Conduct applies to all project spaces:

- GitHub repository (issues, PRs, discussions, code review)
- Security Advisories
- HuggingFace Space community tab
- Any other channel tied to the project (chat, email, social media)

It also applies when an individual is officially representing the project
in public spaces.

## Reporting

If you experience or witness behavior that violates this Code, report
it privately via **GitHub Security Advisories**:

👉 https://github.com/UnlimitedEdition/security-scanner/security/advisories/new

Use the "report a vulnerability" channel — the maintainers will redirect
non-security reports internally. This keeps reports out of public view
and gives the reporter control over disclosure.

### Privacy

Reports are handled confidentially. The reporter will not be identified
to the person being reported without the reporter's explicit consent.

## Enforcement

Maintainers may, at their discretion:

1. **Educate** — ask the contributor to edit or withdraw a comment or PR
2. **Warn** — issue a private warning explaining which behavior was
   unacceptable, with a copy of the relevant Code section
3. **Restrict** — temporarily limit the contributor's access (close PRs,
   lock issues, revoke invitations)
4. **Ban** — permanently remove the contributor from the project

We prefer steps 1 and 2 whenever possible. Escalation is reserved for
repeated or deliberate violations.

### Enforcement timeline

- Reports are acknowledged within **72 hours**
- Investigation completed within **2 weeks**
- Outcomes communicated privately to both parties
