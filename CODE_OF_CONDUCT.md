# Code of Conduct

## Our pledge

We want Web Security Scanner to be a welcoming project for everyone who
wants to contribute — regardless of experience level, background, or
identity. Security tooling only improves when people feel comfortable
reporting bugs, asking questions, and disagreeing about design choices.

## Expected behavior

- **Focus on the work.** Technical disagreement is welcome; personal
  attacks are not. Critique code, not the contributor.
- **Be patient with newcomers.** Everyone started somewhere. A good
  answer to a "basic" question makes the whole project stronger.
- **Give credit.** If someone's idea, PR, or report helped you, say so
  in the commit message or PR description.
- **Respect operator context.** Many users of this scanner are
  non-technical website owners. Docs, error messages, and issue
  responses should meet them where they are.

## Unacceptable behavior

The project exists to help people secure websites. Behavior that makes
contributors or users feel unsafe or unwelcome is off-topic and will be
handled by the maintainers. In short: treat others the way a competent
colleague would treat you on their best day.

## Scope

This Code of Conduct applies to all project spaces — the GitHub repo,
pull requests, issues, security advisories, the HuggingFace Space
community tab, and any other channel tied to the project.

## Reporting

If you experience or witness behavior that violates this Code, report
it privately via **GitHub Security Advisories**:

https://github.com/UnlimitedEdition/security-scanner/security/advisories/new

(Use the "report a vulnerability" channel — the maintainers will
redirect non-security reports internally. This keeps reports out of
public view and gives the reporter embargo control.)

Reports are handled confidentially. The reporter will not be identified
to the person being reported without the reporter's explicit consent.

## Enforcement

Maintainers may, at their discretion:

1. Ask the contributor to edit or withdraw a comment or PR.
2. Issue a private warning explaining which behavior was out of scope.
3. Temporarily restrict the contributor from the project (close PRs,
   lock issues).
4. Permanently ban the contributor from the project.

We prefer step 1 and 2 whenever possible. Escalation is reserved for
repeated or deliberate violations.

## Attribution

This Code of Conduct is inspired by the spirit of the Contributor
Covenant (https://www.contributor-covenant.org) and adapted to fit the
small-team, security-focused nature of this project.
